# Contributing to HIBP

Thank you for your interest in contributing to the **Hyperlocal Intent Broadcast Protocol (HIBP)**.

HIBP is an open, extensible standard. Contributions help improve the protocol, expand registries, and strengthen the ecosystem.

---

## 🧩 Ways to Contribute

You may contribute by:

- Proposing new intent codes  
- Proposing new device types  
- Suggesting transport extensions  
- Improving security or privacy considerations  
- Providing implementation feedback  
- Reporting issues or inconsistencies  
- Submitting pull requests  
- Proposing documentation improvements in `/docs/`  

---

## 📄 Contribution Requirements

All contributions must:

1. Be technically sound  
2. Include clear justification  
3. Maintain backward compatibility unless explicitly proposing a new major version  
4. Follow the formatting conventions used in `/spec/`  
5. Fit within the established domain taxonomy (Commerce, Community, Lost & Found, Ambient, Infrastructure)  
6. Respect reserved ranges and versioning rules  
7. Include updates to registries when applicable  

---

## 🧭 Versioning & Registry Rules

HIBP follows a strict versioning model:

- **v1.x**: Non‑breaking changes only  
- **v2.0**: Required for breaking changes  
- **Registries may expand** without requiring a new protocol version  
- **Reserved ranges MUST NOT be used** without explicit approval  
- **Vendor‑specific and experimental ranges** require clear labeling and justification  

---

## 📝 Submitting Changes

1. Fork the repository  
2. Create a feature branch  
3. Commit your changes with clear messages  
4. Submit a pull request  
5. Reference any related issues  
6. Include test cases or examples when relevant
